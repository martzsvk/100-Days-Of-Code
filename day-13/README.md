# Day 13: Debugging - How to Find and Fix Errors 🐞

Instead of building a new project, I focused on identifying, isolating, and fixing common Python bugs in several broken code snippets.

## What I Learned 🛠️
* **Untangling Logic Errors:** Practiced the "Describe the Problem" technique — explaining what the code should do vs. what it actually does.
* **Reproducing the Bug:** Learning how to consistently trigger an error to understand its cause.
* **The Power of `print()`:** Using strategic print statements to track variable states at different execution points.
* **Using a Debugger:** Stepping through code line-by-line to see exactly where the logic fails.

## The Debugging Process 🧠
1. Describe the Problem (Read the code and understand it)
2. Reproduce the Bug (Find the edge case)
3. Play Computer (Evaluate each line manually)
4. Fix Errors (And test again)
5. Use a Debugger (Thonny or built in debuggers)
